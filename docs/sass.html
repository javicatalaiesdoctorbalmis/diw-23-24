CSS preprocessors

  • Sass is a CSS preprocessor (like Less or PostCSS)
  • CSS preprocessors will add some features to CSS:
    • Variables
    • Nesting
    • Mixins
    • Inheritance
  • More readable and easier to maintain than pure CSS
  • CSS preprocessor code must be compiled into CSS

CSS preprocessors

Sass

  • CSS with superpowers!
  • Free and open-source
  • Compatible with all versions of CSS
  • Many CSS frameworks (e.g. Bootstrap) use Sass
  • Sass has two syntaxes:
    • SCSS syntax (.scss): it's a superset of CSS, and it's used most commonly
    • Idented syntax (.sass): it uses indentation rather than curly braces

Installing Sass

  • The Sass team maintains two libraries for Sass:
    • Pure JavaScript library (a little slower but can be installed on all platforms Node.js supports)
    • Dart Sass library (faster but only supports Windows, Mac OS, and Linux)
  • We will install pure JavaScript library using npm by running this command: 
    npm install -g sass

Learn more about Sass installation here

Preprocessing Sass files

  • You can compile your Sass to CSS using the sass command: 
    • sass input.scss output.css
  • You can re-compile CSS each time you save your Sass: 
    • sass -watch input.scss output.css
  • You can also watch and output to directories: 
    • sass -watch app/sass:app/css

Variables

  • Variables are a way to store information that you want to reuse throughout your stylesheet.
  • You can store things like colors, font stacks, or any CSS value you think you'll want to reuse.
  • Sass uses the $ symbol, followed by a name, to declare and reference variables.

Variables

$myColor: #326dee;
h1 {
color: $myColor;
}
a {
color: $myColor;
}                        
h1 {
    color: #326dee;
 }
  a {
    color: #326dee;
 }

Partials

  • You don't have to write all your Sass in a single file.
  • @import rule loads another Sass file, so you can refer to its members in your Sass file. 
    • @import 'base'; //Without .scss extension
  • A partial is a Sass file named with a leading underscore. It should not be generated into a CSS file.
    • @import '_base'; //Without .scss extension

Partials

//_colors.scss
$myColor: #326dee;
//index.scss
@import 'colors'

h1 {
color: $myColor;
}
a {
color: $myColor;
}

When importing several Sass files, order in @import directives is relevant.

Partials

In October of 2019 Sass announced their new module system, which primarily addressed the switch from using @import to @use instead. However, a lot of Sass build systems doesn't support @use yet (in fact, Bootstrap still uses @import in its Sass code).

Nested rules

  • Sass let you nest your CSS selectors just like in HTML.
  • It is cleaner and easier to read than standard CSS. 
nav {
    ul {
        list-style: none;
    }
    li {
        display: inline-block;
    }
}

nav ul {
    list-style: none;
}
nav li {
    display: inline-block;
}

Nested properties

  • Sass also let you nest CSS properties with the same prefix. 
body {
  background: {
     color: yellow;
     attachment: fixed;
     repeat: no-repeat;
   }
}
body {
  background-color: yellow;
  background-attachment: fixed;
  background-repeat: no-repeat;
}

Parent selector (&)

  • It is a special selector invented by Sass that is used in nested selectors to refer to the outer selector. 
.alert {
    &:hover {
        font-weight: bold;
    }
    :not(&) {
        opacity: 0.8;
    }
}
.alert:hover {
    font-weight: bold;
}
:not(.alert) {
    opacity: 0.8;
}

Built-in modules

  • Sass provides many built-in modules which contain useful functions.
    • math
    • color
    • list
    • map
    • string
    • selector
    • meta
  • These modules can be loaded with the @use rule.

Built-in modules

@use 'sass:math';

$base-font-size: 1rem;

.footnote {
    font-size: math.div($base-font-size, 2);
}
.footnote {
    font-size: 0.5rem;
}
Numeric operators [+,-,*,%] can be used directly in Sass.

Lists and maps

  • A Sass list contains a sequence of other values, separated by commas or spaces.
  • List members can be goruped by brackets or parenthesis, but it's no necessary.
    • $list1: [Helvetica, Arial, sans-serif];
$list2: (10px 12px 10px 20px);
$list3: #fad941, #ffffff, #e02520;

Lists and maps

  • The map data type represents one or more key/value pairs.
  • The keys must be unique, but the values may be duplicated.
  • Unlike lists, maps must be written with parentheses around them.
    • $map: ("regular": 400, "medium": 500, "bold": 700);

Lists an maps

  • We can loop through list and maps members using @each rule.
$sizes: 40px, 80px;

@each $size in $sizes {
  .icon-#{$size} {
    height: $size;
    width: $size;
  }
.icon-40px {
  height: 40px;
  width: 40px;
}
.icon-80px {
  height: 80px;
  width: 80px;
}
Interpolation (#{}) allows you to embed variables into CSS.

Lists an maps

  • We can loop through list and maps members using @each rule.
$colors: (
"blue": #1919e6, 
"red": #e61919);

@each $key, $val in $colors {
    .text-#{$key} {
        color: $val;
    }
}
.text-blue {
    color: #1919e6;
}
.text-red {
    color: #e61919;
}
Lots of functions in list and map built-in modules!

Flow control (@if)

  • Conditionals statements are made using @if rule.
  • An @if rule can optionally have an @else rule.
    • $circle: true;
.avatar {
    @if $circle {
      border-radius: 50px;
    } @else {
        border-radius: 5px;
    }
    .avatar {
    border-radius: 50px;
}
    Relational (==, !=, <,>, <=,>=) and boolean (and, or, not) operators are allowed.

Flow control (@for)

  • @for rule allows us to iterate between two values.
    • @for $i from 1 to 3 {
    .font-size-#{$i} {
        font-size: $i * 10px;
    }
}

    .font-size-1 {
    font-size: 10px; 
}
  
.font-size-2 {
    font-size: 20px; 
}
    By default the last number is excluded, but if we put "through" instead of "to" it will be included.

Mixins

  • A mixin lets you reuse groups of CSS declarations.
  • To create a mixin you use the @mixin directive.
  • You can use it with the @include directive.
    • @mixin important-text {
    font-size: 25px;
    font-weight: bold;
}

.danger {
    @include important-text;
    color: red;
}
    .danger {
    font-size: 25px;
    font-weight: bold;
    color: red;
}

Mixins

  • You can pass in arguments to your mixin.
@mixin important-text($text-color) {
    font-size: 25px;
    font-weight: bold;
    color: $text-color;
}

.danger {
    @include important-text(red);
}
.danger {
    font-size: 25px;
    font-weight: bold;
    color: red;
}

Mixins

  • It's possible to define default values for variables.
@mixin important-text($text-color:red) {
    font-size: 25px;
    font-weight: bold;
    color: $text-color;
}

.danger {
    @include important-text;
}
.danger {
    font-size: 25px;
    font-weight: bold;
    color: red;
}
